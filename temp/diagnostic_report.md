## 3. Требования к исправлению:

### 3.1 Пошаговое редактирование товара:
1. Название (name)
2. Описание (description)
3. Цена с единицами (price + unit_type)
4. Остатки (stock_grams)
5. Единицы измерения (unit_type + measurement_step)
6. Изображение (image_url)
7. Категория (category_id)

### 3.2 Логика:
- Для каждого шага: "Хотите изменить [параметр]? (да/нет)"
- Если "да" → запрос нового значения
- Если "нет" → переход к следующему шагу
- В конце: сохранение всех изменений

### 3.3 Система бэкапов:
- Автоматический запуск в 4:00 по сербскому времени
- Хранение только 10 последних бэкапов каждого файла
- Автоматическое удаление старых
